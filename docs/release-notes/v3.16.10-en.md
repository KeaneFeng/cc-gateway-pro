# CC-Gateway-Pro v3.16.10

> Proxy request log viewer, opt-in Session Traces, safer local trace defaults, and refreshed documentation/screenshots.

## Highlights

- **View Logs**: open the last 7 days of proxy request logs in your preferred terminal with real-time scrolling.
- **Session Traces**: off by default, records only after you enable it, and keeps data local.
- **Capture modes**: Summary stores extracted fields, context categories, tool calls, per-turn usage, and response previews; Full stores redacted request/response JSON for deeper debugging.
- **Trace safety defaults**: sensitive keys are redacted, default retention is 14 days, and response previews are limited to 2000 characters.
- **Docs and previews**: README and docs now cover the new workflow; interface previews use demo paths/URLs and consistent no-black-border SVG sizing.

## Usage Notes

Use **Settings → Advanced → Log Configuration → View Logs** when you want to watch proxy traffic live while debugging routing, status codes, model selection, provider selection, latency, or errors.

Use **Session Traces** only when you need context analysis. Start with Summary mode. Use Full mode only on trusted devices because it stores more request/response detail, even though sensitive keys are redacted.
