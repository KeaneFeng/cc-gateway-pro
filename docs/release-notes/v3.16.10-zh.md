## 重点内容

- **View Logs**：在偏好的终端中实时滚动查看最近 7 天的代理请求日志。
- **Session Traces**：默认关闭，只有用户显式开启后才记录，数据保存在本机。
- **采集模式**：Summary 保存提取字段、上下文分类、工具调用、每轮 usage 和响应预览；Full 保存脱敏后的 request/response JSON，适合深度排查。
- **Trace 安全默认值**：敏感 key 会脱敏，默认保留 14 天，响应预览限制为 2000 字符。
- **文档与预览图**：README 和 docs 已补充新流程说明；界面预览统一为 demo 路径/URL，移除黑边并保持 SVG 尺寸一致。

## 使用提示

排查路由、状态码、模型选择、Provider 选择、延迟或错误时，可以通过 **设置 → 高级 → 日志配置 → View Logs** 实时观察代理请求流。

需要分析上下文压力、工具/Skills/MCP 使用情况和每轮 usage 时，再开启 **Session Traces**。建议优先使用 Summary 模式；Full 模式虽然会脱敏敏感 key，但会保存更多请求/响应细节，请只在可信设备上启用。
