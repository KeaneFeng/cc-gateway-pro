## ハイライト

- **View Logs**：好みのターミナルで過去 7 日分のプロキシリクエストログをリアルタイム表示。
- **Session Traces**：既定ではオフ。ユーザーが明示的に有効化した後だけ記録し、データはローカルに保存。
- **キャプチャモード**：Summary は抽出フィールド、コンテキスト分類、ツール呼び出し、ターンごとの usage、レスポンスプレビューを保存。Full はマスク済み request/response JSON を保存し、詳細調査に使用。
- **Trace の安全な既定値**：機密 key をマスクし、既定の保持期間は 14 日、レスポンスプレビューは 2000 文字に制限。
- **ドキュメントとプレビュー**：README と docs に新しいワークフローを追加。画面プレビューは demo パス/URL に統一し、黒枠を削除して SVG サイズを揃えました。

## 使用メモ

ルーティング、ステータスコード、モデル選択、Provider 選択、レイテンシ、エラーを調査するときは、**設定 → 詳細 → ログ設定 → View Logs** からプロキシリクエストをリアルタイムに確認できます。

コンテキスト圧力、ツール/Skills/MCP 使用状況、ターンごとの usage を分析したい場合だけ **Session Traces** を有効化してください。まずは Summary モードを推奨します。Full モードは機密 key をマスクしますが、より多くの request/response 詳細を保存するため、信頼できるデバイスでのみ使用してください。
